## Delivery Modes at a Glance

| Mode | What It Is | Runtime | Guardrails | Best For |
|------|-----------|---------|------------|----------|
| **Plugin** | Distributable VS Code extension | VS Code Copilot / Claude Code | Prompt-embedded (rigor inlined into agents) | Teams, shared tooling, Marketplace distribution |
| **Copilot Agent** | Repo-local `.github/agents/` config files | VS Code Copilot / Claude Code | Prompt-embedded (rigor inlined into agents) | Per-project customization, checked into source control |
| **Markdown** | Platform-agnostic `.md` spec files | Any LLM (ChatGPT, Gemini, Ollama, etc.) | Documented only (`guardrails.md`) | Maximum portability, no platform dependency |
| **Fullstack** | Python package (`BaseScientificAgent` subclass) | CLI, Web UI, MCP server | Middleware-enforced (CodeScanner, BoundsChecker) | Custom Python integrations, programmatic agents |

---

> **Plugin vs Copilot Agent вҖ” what's the difference?**
>
> - **Plugin**: A distributable VS Code extension (Marketplace or local install). Ships compiled agents, skills, and prompt modules as a self-contained package. Install once, works across every workspace. Think of it as a portable toolbox.
> - **Copilot Agent**: Repo-local `.github/agents/` configuration files generated by the wizard for a specific project. Checked into your repository, automatically discovered by VS Code Copilot. Also generates `.claude/agents/` for Claude Code. Think of it as project-specific customization.

---

Each agent is a Markdown file with YAML frontmatter defining its `tools`, `handoffs`, and `instructions` references. The coordinator inspects the user's request and hands off to the appropriate specialist. Prompt modules (scientific rigor, code execution best practices) are inlined during the build вҖ” agents carry their guardrail instructions with them.

**Profiles** control which agents are generated:
- **Full** вҖ” all 9 agents (coordinator, coder, data-qc, report-writer, rigor-reviewer, code-reviewer, analysis-planner, docs-ingestor, domain-assembler)
- **Compact** вҖ” agents merged into fewer roles (e.g., code-reviewer + rigor-reviewer вҶ’ reviewer)
- **Custom** вҖ” user-selected subset

**Dual-platform** вҖ” the wizard generates both `.github/agents/` (VS Code Copilot) and `.claude/agents/` (Claude Code) from the same `WizardState`.
